API

你可以用web-ifc-three做的很棒的事情列表#

📃 所有的API都有记录，所以当你使用本文档中看到的任何对象或方法时，你应该在Intellisense中看到帮助，无论你使用的是哪种IDE。看看吧！。

👩‍🏫 然而，我们意识到，阅读intellisense或注释并不是最舒服的事情，所以在这一页，我们将对API能做什么做一个概述。一切都将在以后的具体教程中得到更详细的介绍。

IfcLoader#

💎 这是我们将从库中导入的唯一对象。它包含了使用IFC所需的所有逻辑。你可以使用它的load()和loadAsync()方法从URL加载IFC，就像其他Three.js加载器一样。例如，要加载一个IFC，我们可以做以下工作。

import { IFCLoader } from "web-ifc-three/IFCLoader";
const ifcLoader = new IFCLoader();
ifcLoader.load(
"models/Example_model.ifc",
(ifcModel) => scene.add(ifcModel));

🏠🏠🏠 通过web-ifc-three，你可以在场景中加载多个模型。许多API操作是在一个指定的模型上执行的。为了表达我们想在哪个模型上操作，我们必须给出它的ModelID。

🔍 你可以通过生成的网格的属性modelID获得模型的ID。

const modelID = ifcModel.modelID;

✌ 有两种方法可以访问API。

  • 通过IfcLoader用load和loadAsync生成的IfcModel的实例。

  • 通过IfcLoader.IfcManager。

💡
IfcModel实例和IfcLoader.IfcManager都可以互换使用，以访问API。有一个区别：当使用IfcModel时，ModelID不必作为一个参数给出（它是隐含的）。

🧰 设置#

setWasmPath#

async IfcLoader.IfcManager.setWasmPath (
path: string
): void;

🧠 指定web-ifc.wasm和web-ifc-mt.wasm文件的位置。这些文件是web-ifc文件，使用IFC.js创建任何应用程序都需要它们。你可以在node_modules/web-ifc/中找到它们。

🚧

小心你使用的工具!

如果你使用React、Angular、Vue或Svelte等框架或库，那么项目的根路径有可能与被服务的应用程序的根路径不一致。你必须在每种情况下检查如何管理静态服务文件的路径。

函数的参数。#

  • path 文件web-ifc.wasm的路径。

例子。#

如果web-ifc.wasm在dist/wasmDir中。

await ifcLoader.setWasmPath("dist/wasmDir/");

setupThreeMeshBVH#

IfcLoader.IfcManager.setupThreeMeshBVH (
computeBoundsTree: any,
disposeBoundsTree: any,
acceleratedRaycast: any
): void;

⚡⚡⚡ 这种方法可以使对象的拾取速度更快，特别是对于具有重度几何形状的非常大的模型。这个方法实际上允许IFC.js使用Garrett Johnson的惊人的库。你可以用npm i three-mesh-bvh或yarn add three-mesh-bvh来安装它。

☝ 使用这种方法不是强制性的，但如果你想在60帧的情况下能够选择中型/大型IFC模型中的对象，这是必要的。

函数的参数。#

  • computeBoundsTree 预制的BufferGeometry扩展函数，它建立了一个新的BVH，将其分配给boundsTree，并将新的索引缓冲区应用到几何体上。- 来源

  • disposeBoundsTree BufferGeometry扩展函数，用于处置BVH。- 来源。

  • acceleratedRaycast 与THREE.Mesh.raycast相同的加速射线传输函数。如果BVH可用的话，就使用它来进行光线投射，否则就回到内置方法。- 来源。

例子。#

import {
IFCLoader
} from 'web-ifc-three/dist/IFCLoader';
import {
acceleratedRaycast,
computeBoundsTree,
disposeBoundsTree
} from 'three-mesh-bvh';
const ifcLoader = new IFCLoader();
ifcLoader.ifcManager.setupThreeMeshBVH(
acceleratedRaycast,
computeBoundsTree,
disposeBoundsTree);

setOnProgress#

IfcLoader.IfcManager.setOnProgress (
onProgress: (event: ParserProgress) => void
): void;

⏳ 设置一个回调函数，每加载10%的IFC都会被调用。这样，你可以显示一个加载条，向你的用户展示加载的情况。

函数的参数。#

  • onProgress: 回调函数，每10%被调用。该函数必须有一个参数，该参数接收一个具有两个属性的对象。loaded（加载的对象数量）和total（文件中的对象数量）。

例子。#

function exampleCallback(event) {
const progress = event.total / event.progress * 100;
console.log("Progress: ", progress, "%");
}
ifcLoader.ifcManager.setOnProgress(exampleCallback);

applyWebIfcConfig#

async IfcLoader.IfcManager.applyWebIfcConfig (
settings: LoaderSettings
): void;

🎹🎹🎹 应用web-ifc的配置，它是这个库内部使用的分析器。

函数的参数。#

  • settings: 一个包含以下字段的对象。

  • COORDINATE_TO_ORIGIN: boolean: 是否将模型带到场景的中心。对地理定位的模型有用。

  • USE_FAST_BOOLS: boolean: 是否使用更快的（不那么精确的）布尔逻辑。

  • BOOL_ABORT_THRESHOLD?: number: 布尔运算计算的极限。

  • CIRCLE_SEGMENTS_LOW?: number: 低段曲线的分辨率。

  • CIRCLE_SEGMENTS_MEDIUM?: number: 中段曲线的分辨率（例如IfcSweptDiskSolid）。

  • CIRCLE_SEGMENTS_HIGH?: number: 高段曲线的分辨率（如IfcCircle）。

例子。#

如果一个文件是地理定位的，我们想把它带到场景的原点。

await ifcLoader.ifcManager.applyWebIfcConfig({
COORDINATE_TO_ORIGIN: true,
USE_FAST_BOOLS: false
});

useWebworkers#

async IfcLoader.IfcManager.useWebWorkers (
active: boolean,
path?: string
): void;

👷‍♀️👷‍♀️👷‍♀️ 切换包含解析逻辑的网络工作者。API的其余部分保持不变，但所有的逻辑将发生在机器的另一个线程中，因此应用程序永远不会被繁重的操作所阻断。

🚧 你需要将文件IFCWorker.js复制到你的项目中，并将相对路径作为第二个参数传递。

函数的参数。#

  • active: 是否使用网络工作者。
  • path: 工作器文件的相对路径。如果 active = true，这是必要的。

例子。#

如果IFCWorker.js是在一个叫做: files。

await ifcLoader.ifcManager.useWebWorkers({
true,
"files/IFCWorker.js"
});

useJSONData#

async IfcLoader.IfcManager.useJSONData (
useJSON: boolean
): void;

🚀 使用JSON属性数据，而不是WASM数据，它消耗的内存要少得多。只在以下情况下使用。

  • 如果你不需要访问IFC的属性
  • 如果你将提供JSON格式的属性。 你可以按照这个例子提前从IFC文件中获取JSON。

函数的参数。#

  • useJSON: 是否使用JSON数据。

例子。#

await ifcLoader.ifcManager.useJSONData(true);

addModelJSONData#

async IfcLoader.IfcManager.addModelJSONData (
modelID: number,
data: { [id: number]: JSONObject
): void;

📦📦📦 将一个模型的属性作为JSON数据添加。如果你使用网络工作者，请使用loadJsonDataFromWorker()代替，以避免开销。

函数的参数。#

  • modelID: IFC模型的ID。
  • data: 数据是一个对象，其中键是expressIDs，值是属性。此外，每个值都有属性expressID（项目的id）和type（项目的IFC类别）。

例子。#

await ifcLoader.ifcManager.addModelJSONData(0, jsonData);

loadJsonDataFromWorker#

async IfcLoader.IfcManager.loadJsonDataFromWorker (
modelID: number,
path: string
): void;

📦📦📦 从JSON文件中直接从Web Worker加载IFC模型的数据。如果你不使用网络工作者，请使用addModelJSONData()代替。

函数的参数。#

  • modelID: IFC模型的ID。
  • path: JSON文件的路径 相对于网络工作者的文件。

例子。#

await ifcLoader.ifcManager.loadJsonDataFromWorker(0, "path/to/data.json");

🔍 获取器#

getExpressId#

IfcLoader.IfcManager.getExpressId (
geometry: BufferGeometry,
faceIndex: number
): number;

💳 从一个面的索引中获取IFC元素的快速ID。

📜

为什么要有脸部指数？

因为当我们用鼠标在三维空间中选择一个物体时，我们会得到该面的索引。我们通常希望得到该面所属物体的ID，以便高亮显示它，隔离它或获得其所有属性。

函数的参数。#

  • geometry 你用鼠标点击的模型的Geometry。

  • faceIndex 与raycaster相交的面的索引。如果你不熟悉raycaster，不要担心，我们会在关于picking的教程中详细介绍这个。

例子。#

const intersected = raycaster.intersectObject(mesh)[0];
const index = intersected.faceIndex;
const id = ifcLoader.ifcManager.getExpressId(mesh, index);

getIfcType#

IfcLoader.IfcManager.getIfcType (
modelID: number,
id: number,
): string;

🎸🪕🎻 获取指定元素的IFC类型（例如：IFCWALL）。

💡
IFC中的元素总是有一个相关的类型。IfcWall, IfcSlab, IfcWindow, IfcDoor，等等。

函数的参数。#

  • modelID IFC模型的ID。

  • id 要获取其属性的项目的Express ID。你可以通过getExpressId（如果你在3D视图中挑选一个对象）或者通过getAllItemsOfType或getSpatialStructure遍历模型来获得。

例子。#

const model = ifcModel.modelID;
const id = 2142;
const manager = loader.ifcLoader.ifcManager;
const type = manager.getIfcType(model, id);

getAllItemsOfType#

async IfcLoader.IfcManager.getAllItemsOfType (
modelID: number,
type: number,
verbose: boolean
): number[] | object[];

🎸🎸🎸 返回一个特定模型的所有IFC类型的对象（例如，所有墙壁、所有地板、所有窗户等）。它可以返回一个expressIDs'数组，或者（如果verbose = true'）一个包含找到的项目的属性的对象数组。

函数的参数。#

  • modelID IFC模型的ID。

  • type 你想得到的元素的IFC类型。你可以直接从web-ifc导入这些类型（见下面的例子）。

  • verbose 如果是真的，则获得所有找到的项目的属性。要小心，因为这在较大的模型中可能是一个缓慢的操作。

例子。#

import { IFCWALLSTANDARDCASE as W } from 'web-ifc';
const manager = loader.ifcLoader.ifcManager;
const walls = await manager.getAllItemsOfType(0, W, false);

getItemProperties#

async IfcLoader.IfcManager.getItemProperties (
modelID: number,
id: number,
recursive = false
): object[];

📕🔍 获取给定元素的本地属性。在IFC模式中，有两种类型的属性：直接和间接。如何获得间接属性（psets、qsets和类型属性）将在后面讨论。

💡
所有与属性有关的方法都返回一个对象数组，其中键是属性的名称，值是它们的值。

函数的参数。#

  • modelID IFC模型的ID。

  • id 要获取其属性的项目的Express ID。你可以通过getExpressId（如果你在3D视图中挑选一个对象）或者通过getAllItemsOfType或getSpatialStructure遍历模型来获得。

  • recursive 如果是真的，就会递归地获得所有被引用的元素的属性。要小心，因为这在较大的模型中可能是一个缓慢的操作。

例子。#

const model = ifcModel.modelID;
const id = 2142;
const manager = loader.ifcLoader.ifcManager;
const props = await manager.getItemProperties(model, id, false);

getPropertySets#

async IfcLoader.IfcManager.getPropertySets (
modelID: number,
id: number,
recursive = false
): object[];

📗🔍 获取给定元素的属性集和数量集。

📃

财产集？

本地和类型属性是那些由IFC模式预定义的属性：它们总是包含相同的信息。另一方面，属性集则是任意的，可以由用户定义。

函数的参数。#

  • modelID IFC模型的ID。

  • id 要获取其属性的项目的Express ID。你可以通过getExpressId（如果你在3D视图中挑选一个对象）或者通过getAllItemsOfType或getSpatialStructure遍历模型来获得。

  • recursive 如果是真的，就会递归地获得所有被引用的元素的属性。要小心，因为这在较大的模型中可能是一个缓慢的操作。

例子。#

const model = ifcModel.modelID;
const id = 2142;
const manager = loader.ifcLoader.ifcManager;
const props = await manager.getPropertySets(model, id, false);

getSpatialStructure#

async IfcLoader.IfcManager.getSpatialStructure (
modelID: number
): object;

🔗🔗🔗 获取项目的空间结构。

💡
The spatial structure is the hierarchical structure that organizes every IFC project (all physical items are referenced to an element of the spatial structure).

函数的参数。#

  • modelID IFC模型的ID。

例子。#

const model = ifcModel.modelID;
const manager = loader.ifcLoader.ifcManager;
const ifcProject = await manager.getSpatialStructure(model);

🧱 子集#

getSubset#

IfcLoader.IfcManager.getSubset (
modelID: number,
material?: Material
): object;

🧱🔍 获取指定材质的子集的网格。如果没有给定材质，则返回具有原始材质的子集。

💡
几何子集是对模型的几何形状的提取。例如，你可以提取所有符合某个条件的IfcDoors和IfcWindows的子集，并突出显示或导出它们。

函数的参数。#

  • modelID IFC模型的ID。

  • material (可选）分配给子集的材料（如果有的话）。

例子。#

const model = ifcModel.modelID;
const manager = loader.ifcLoader.ifcManager;
const subset = manager.getSubset(model);

createSubset#

IfcLoader.IfcManager.createSubset (
config: SubsetConfig
): object;

🧱✨ 创建一个新的几何子集。

函数的参数。#

  • config 一个具有下列选项的配置对象。

    • scene 模型所在的场景。

    • modelID 该模型的ID。

    • ids 将符合子集的模型的项目的ExpressIDs。

    • removePrevious 是否要用这个材料删除这个模型的前一个子集。

    • material (可选）应用于子集的材料。如果没有给出材料，子集就有原始材料。

    • customID (可选） 可选的自定义名称，用于给子集命名。这允许用相同的材料创建多个子集。

例子。#

const model = ifcModel.modelID;
const manager = loader.ifcLoader.ifcManager;
const config = {
modelID: model
scene: scene,
ids: [id],
removePrevious: true
}
manager.createSubset(config);

removeSubset#

IfcLoader.IfcManager.removeSubset (
modelID: number,
material?: Material,
customID?: string,
): object;

🧱💣 移除指定的几何子集。

函数的参数。#

  • modelID IFC模型的ID。

  • material (可选）分配给子集的材料（如果有的话）。

  • customID (可选) 可选的自定义名称，用于给子集命名。这允许用相同的材料创建多个子集。

例子。#

const model = ifcModel.modelID;
const manager = loader.ifcLoader.ifcManager;
manager.removeSubset(model);

removeFromSubset#

IfcLoader.IfcManager.removeSubset (
modelID: number,
ids: number[],
customID?: string,
material?: Material
): void;

🧱💥🤏 从指定的几何子集中删除指定的项目。

函数的参数。#

  • modelID IFC模型的ID。

  • ids 将符合子集的模型的项目的ExpressIDs。

  • material (可选）分配给子集的材料（如果有的话）。

  • customID (可选) 可选的自定义名称，用于给子集命名。这允许用相同的材料创建多个子集。

例子。#

import { IFCWALLSTANDARDCASE as W } from 'web-ifc';
const model = ifcModel.modelID;
const manager = loader.ifcLoader.ifcManager;
const walls = await manager.getAllItemsOfType(0, W, false);
manager.removeFromSubset(model, walls);

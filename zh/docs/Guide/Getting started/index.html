开始工作

了解IFC.js#

🐟🐠🐡 IFC.js不仅仅是一个库：它是一个库和项目的生态系统，使创建BIM工具变得微不足道。这种模块化的原因有两个。

  • 为了能够在我们的应用程序中只使用我们需要的东西。
  • 为了使库更容易维护。

了解它是由哪些部分组成的很重要，以便知道在每种情况下应该使用什么。IFC.js本质上由3层组成，每层都有独特的责任。

🧩

web-ifc

一个IFC文件分析器。它能够从IFC中读取所有信息，编辑它并写入新文件。它没有3D浏览器，只对数据起作用。
何时使用。
当你想读或写IFC文件时，不需要任何查看器。这个库提供了对数据的全部权力，但需要对IFC模式有更多的了解才能使用它。
视情况而定。 -

🧩

web-ifc-three

一个3D BIM查看器。它允许以60帧/秒的速度查看和浏览三维模型，选择元素并轻松读取所有的IFC数据以显示给终端用户。它是three.js的官方IFC加载器。
何时使用。
当你想为你的应用程序制作一个BIM查看器，并且你想完全控制所有实现的功能。
视情况而定。 web-ifc

🧩

web-ifc-viewer

一个三维BIM浏览器，已经实现了许多工具和功能（剖面图、尺寸等），允许你用很少的努力来创建BIM工具。
何时使用。
=当你想创建一个BIM浏览器，而你又不想花时间实现所有你想拥有的模型导航工具。
视情况而定。 web-ifc-three

🧠 メモリ

はじめに#

📭📫 多くの場合、BIMビューアを開いたり閉じたりできるアプリケーションを作りたいでしょう。 当たり前のように思われるかもしれませんが、問題があります。メモリ管理 です。

🤔 さて、Webアプリケーションではメモリ管理は一般的な問題ではないので、多くの人が頭を悩ませていることでしょう。JavaScriptやその他のモダンな言語には、自動メモリ管理/ガベージコレクタがあり、プログラマはオブジェクトや配列を解放することを心配する必要はありません。

✍🤖❓ However, when working with Three.js, data going to the graphics card (e.g. buffers) is not affected by this automatic memory management.

🤕 つまり、Three.jsで作ったビューアを閉じると（IFC.jsの有無に関わらず）、💦 メモリリーク が発生するのです。つまり、ユーザーのRAMメモリの一部をブロックしているのです。リークが大きくなりすぎると、アプリケーションの動作がどんどん遅くなり、クラッシュする可能性があります。

💡
このトピックに関する詳しい情報はこちらでご覧いただけます。

🚀 例えば、ReactやAngularなどのフレームワークやライブラリを使ってSPA（Single Page Application）を作成している場合、これは特に重要になります。このような場合、ウェブアプリケーションが再読み込みされることはなく、メモリリークが蓄積されていきます。

幸いなことに、IFC.jsはこの点を考慮し、使用済みメモリを非常に簡単に解放することができます。では、その方法を見てみましょう。👇

このチュートリアルの完全な例は、こちらでご覧いただけます。

方法#

メモリの監視#

👀 Three.jsの最適化されたシーンをあまり扱ったことがない人が最初に抱く疑問は、どのようにしてアプリケーションがどれくらいのメモリを消費しているかを見るかということでしょう。そうでなければ、メモリリークを検出することは不可能です。

🌍 これにはいくつかの方法があります。1つ目は、使用しているブラウザの開発者向けツールを使用することです。例えば、Google Chromeには「メモリ」という項目があり、スナップショットを撮って、現在のタブがどれだけのメモリを消費しているかを確認することができます。

🤖 しかし、より便利な方法は、ライブラリ stats.js を利用することです。このライブラリは、Three.jsアプリケーションのパフォーマンスとメモリ**を監視することができます。npm i stats.js` でインストールでき、以下のように使用できます。

// 統計情報
const stats = new Stats();
stats.showPanel(2);
document.body.append(stats.dom);
// アニメーションのループ
const animate = () => {
stats.begin();
// その他をアップデート
stats.end();
requestAnimationFrame(animate);
};
animate();

📋 これにより、実行中のシーンの統計情報が表示されます。これらをクリックすることで、アプリのパフォーマンスとメモリ使用量を切り替えることができます。

ディスポーザブルメモリ#

✌ では、IFC.jsが消費するメモリをすべて削除する関数を作ってみましょう。注意する点は2つあります。

  • 以前にWASMファイルへのパスを指定した場合、web-ifc-threeの新しいインスタンスで再度指定する必要があります。

  • これは重要です。ロードしたIFCモデルへの参照を配列、オブジェクト、またはクラスに保存した場合、それらを手動で削除する必要があります。さもなければ、メモリは解放されません。

async function releaseMemory() {
// これにより、すべてのIFCLoaderのメモリが解放されます。
await ifcLoader.ifcManager.dispose();
ifcLoader = null;
ifcLoader = new IFCLoader();
// 以前にwasmのパスが設定されていた場合、
// それをリセットする必要があります
await ifcLoader.ifcManager.setWasmPath('../../../');
// IFCモデルが配列やオブジェクトである場合は、
// そこでも解放する必要があります。
// そうしないと、ガベージコレクションされません。
models.length = 0;
}

簡単なGUIを設定する#

💅 最後に、HTMLボタンを作成し、先ほど作成した関数にリンクさせます。

<input type="button"
id="memory-button" value="Release memory">
// メモリ廃棄の設定
const button = document.getElementById('memory-button');
button.addEventListener(`click`, () => releaseMemory());

😎 アプリケーションはこのような感じです。

モデルを読み込んで、メモリを解放してみてください。元に戻るのがわかると思います。ブラウザのガベージコレクタには時間がかかるので、数秒かかることがありますので注意してください。また、ブラウザの開発者ツールを使って、手動でガベージコレクタを適用することもできます。

SPA#

🌈 SPAやSingle Page Applicationは、バックエンドとの通信が少なく、効率的であるため、非常に一般的なアプリケーションです。これらのアプリケーションを作るツールの例としては、React、Angular、Vueなどがあります。

👶👴 多くの場合、これらのツールは、アプリケーションコンポーネントのライフサイクルを自動的に管理します。例えば、ユーザーが3Dビューを閉じたとき、そのビューを含むHTML要素は自動的に破棄されます。

🚧🚧🚧 メモリリークを防ぐには、3Dビューを含むコンポーネントが破壊されるたびに、このチュートリアルで見たようにアプリケーションメモリを解放する必要があります。多くの場合、コンポーネントが破壊されるたびに、このロジックを自動的に実行するフックまたは同様のメカニズムがあります。

API

web-ifc-threeでできるすごいことリスト#

📃 すべてのAPIはドキュメント化されていますので、このドキュメントで紹介されているオブジェクトやメソッドを使用すると、使用しているIDEにかかわらず、Intellisenseでヘルプが表示されるはずです。チェックしてみてください！

👩‍🏫 しかし、インテリセンスやコメントを読むことは、最も快適なことではないことを理解しているので、このページでは、APIでできることの概要を説明します。すべての内容は、後ほど個別のチュートリアルで詳しく説明します。

IfcLoader#

💎 ライブラリーからインポートする唯一のオブジェクトです。IFCを扱うために必要なすべてのロジックが含まれています。そのload()メソッドとloadAsync()メソッドを使用して、URLからIFCを読み込むことができます。他のThree.js Loader と同様です。例えば、IFCを読み込むには以下のようにします。

import { IFCLoader } from "web-ifc-three/IFCLoader";
const ifcLoader = new IFCLoader();
ifcLoader.load(
"models/Example_model.ifc",
(ifcModel) => scene.add(ifcModel.mesh));

🏠🏠🏠 web-ifc-threeでは、シーン内に複数のモデルを読み込むことができます。APIの操作の多くは、指定されたモデルに対して実行されます。どのモデルを操作したいかを表すには、そのモデルのModelIDを指定する必要があります。

🔍 モデルのIDは、生成されたメッシュのプロパティmodelIDで取得できます。このプロパティをThree.js meshに追加します。

const modelID = ifcModel.mesh.modelID;

✌ このAPIにアクセスするには2つの方法があります。

  • IfcLoaderがloadとloadAsyncで生成するIfcModelのインスタンスを介して。

  • IfcLoader.IfcManagerを介して。

💡
IfcModelのインスタンスとIfcLoader.IfcManagerの両方は、APIにアクセスするために互換的に使用することができます。IfcModelを使用する場合、ModelIDを引数として与える必要はありません(暗黙的に知られています)。

🧰 セットアップ#

setWasmPath#

IfcLoader.IfcManager.setWasmPath (
path: string
): void;

🧠 web-ifc.wasmファイルの場所を指定します。このファイルはweb-ifcファイルで、IFC.jsを使ってアプリケーションを作成する際に必要となります。node_modules/web-ifc/web-ifc.wasmにあります。

🚧

使う道具には気をつけよう

React、Angular、Vue、Svelteなどのフレームワークやライブラリを使用している場合、プロジェクトのルートパスと提供されるアプリケーションのルートパスが一致しないことがあります。静的に提供されるファイルのパスがどのように管理されているか、各ケースで確認する必要があります。

関数の引数#

  • path web-ifc.wasmのルートです。

例#

web-ifc.wasm が dist/wasmDir にある場合。

ifcLoader.setWasmPath("dist/wasmDir/");

setupThreeMeshBVH#

ifcLoader.IfcManager.setupThreeMeshBVH (
computeBoundsTree: any,
disposeBoundsTree: any,
acceleratedRaycast: any
): void;

⚡⚡⚡ この方法により、オブジェクトのピッキングが格段に速くなり、特に重いジオメトリを持つ非常に大きなモデルの場合に有効です。この方法により、IFC.jsはGarrett Johnson氏の素晴らしいライブラリを使用することができます。npm i three-mesh-bvhまたは yarn add three-mesh-bvh でインストールできます。

この方法は必須ではありませんが、中/大IFCモデルのオブジェクトを60fpsで選択できるようにしたい場合は必要です。

関数の引数#

  • computeBoundsTree 新しいBVHを構築し、それをboundsTreeに割り当て、新しいインデックスバッファをジオメトリに適用する、プリメイドのBufferGeometry拡張関数です。- ソース。

  • disposeBoundsTree BVHを破棄するBufferGeometry拡張関数です。- ソースを参照してください。

  • acceleratedRaycast THREE.Mesh.raycastと同じシグネチャーを持つ、高速化されたレイキャスト関数です。BVHが利用可能な場合はレイキャスティングに使用し、そうでない場合はビルトインのアプローチに戻ります。- ソースを参照してください。

例#

import {
IFCLoader
} from 'web-ifc-three/dist/IFCLoader';
import {
acceleratedRaycast,
computeBoundsTree,
disposeBoundsTree
} from 'three-mesh-bvh';
const ifcLoader = new IFCLoader();
ifcLoader.ifcManager.setupThreeMeshBVH(
acceleratedRaycast,
computeBoundsTree,
disposeBoundsTree);

🔍 ゲッターズ#

getExpressId#

IfcLoader.IfcManager.getExpressId (
geometry: BufferGeometry,
faceIndex: number
): number;

💳 IFC要素のExpress IDをフェースインデックスから取得する。

📜

なぜ顔認証なのか？

なぜなら、3D空間のオブジェクトをマウスで選択すると、その面のインデックスが得られるからです。通常、その面をハイライトしたり、分離したり、すべてのプロパティを取得するためには、その面が属するオブジェクトのIDが必要です。

関数の引数#

  • geometry マウスでクリックしたモデルの ジオメトリ です。

  • faceIndex raycasterで交差した顔のインデックスです。レイキャスターに慣れていない方もご安心ください。ピッキングについてのチュートリアルで詳しく説明します。

例#

const intersected = raycaster.intersectObject(mesh)[0];
const index = intersected.faceIndex;
const id = ifcLoader.ifcManager.getExpressId(mesh, index);

getIfcType#

IfcLoader.IfcManager.getIfcType (
modelID: number,
id: number,
): string;

🎸🪕🎻 指定された要素のIFCタイプを取得します（例：IFCWALL）。

💡
IFCの要素には、必ず関連するタイプがあります。IfcWall、IfcSlab、IfcWindow、IfcDoorなどです。

関数の引数#

  • modelID IFCモデルのIDです。

  • id プロパティを取得したいアイテムのエクスプレスID。これは、getExpressId (3Dビューでオブジェクトを選択している場合)、または getAllItemsOfType や getSpatialStructure でモデルをトラバースすることで取得できます。

例#

const model = ifcModel.mesh.modelID;
const id = 2142;
const manager = loader.ifcLoader.ifcManager;
const type = manager.getIfcType(model, id);

getAllItemsOfType#

IfcLoader.IfcManager.getAllItemsOfType (
modelID: number,
type: number,
verbose: boolean
): number[] | object[];

🎸🎸🎸 指定されたモデルの指定されたIFCタイプのすべてのオブジェクト（すべての壁、すべての床、すべての窓など）を返します。これは、expressIDの配列を返すか、（verbose = trueの場合）見つかったアイテムのプロパティを含むオブジェクトの配列を返すことができます。

関数の引数#

  • modelID IFCモデルのIDです。

  • type 取得したい要素のIFCタイプです。これらのタイプは web-ifc から直接インポートすることができます (以下の例を参照)。

  • verbose trueの場合、見つかったすべてのアイテムのプロパティを取得します。大きなモデルでは、この処理に時間がかかることがあるので、注意が必要です。

例#

import { IFCWALLSTANDARDCASE as W } from 'web-ifc';
const manager = loader.ifcLoader.ifcManager;
const walls = manager.getAllItemsOfType(0, W, false);

getItemProperties#

IfcLoader.IfcManager.getItemProperties (
modelID: number,
id: number,
recursive = false
): object[];

📕🔍 与えられた要素のネイティブ・プロパティを取得します。IFCスキーマでは、プロパティには直接プロパティと間接プロパティの2種類があります。間接的なプロパティ（psets、qsets、typeプロパティ）の取得方法については後述します。

💡
プロパティに関連するすべてのメソッドは、キーがプロパティの名前で、値がその値であるオブジェクトの配列を返します。

関数の引数#

  • modelID IFCモデルのIDです。

  • id プロパティを取得したいアイテムのエクスプレスID。これは、getExpressId (3Dビューでオブジェクトを選択している場合)、または getAllItemsOfType や getSpatialStructure でモデルをトラバースすることで取得できます。

  • recursive trueの場合、参照されているすべての要素のプロパティを再帰的に取得します。大きなモデルでは、この処理に時間がかかることがありますので、注意してください。

例#

const model = ifcModel.mesh.modelID;
const id = 2142;
const manager = loader.ifcLoader.ifcManager;
const props = manager.getItemProperties(model, id, false);

getPropertySets#

IfcLoader.IfcManager.getPropertySets (
modelID: number,
id: number,
recursive = false
): object[];

📗🔍 与えられた要素のプロパティセットと数量セットを取得します。

📃

Property sets?

ネイティブプロパティとタイププロパティは、IFCスキーマであらかじめ定義されているもので、常に同じ情報を含んでいます。一方、プロパティセットは任意のもので、ユーザーが定義することができます。

関数の引数#

  • modelID IFCモデルのIDです。

  • id プロパティを取得したいアイテムのエクスプレスID。これは、getExpressId (3Dビューでオブジェクトを選択している場合)、または getAllItemsOfType や getSpatialStructure でモデルをトラバースすることで取得できます。

  • recursive trueの場合、参照されているすべての要素のプロパティを再帰的に取得します。大きなモデルでは、この操作に時間がかかることがあるので、注意が必要です。

例#

const model = ifcModel.mesh.modelID;
const id = 2142;
const manager = loader.ifcLoader.ifcManager;
const props = manager.getPropertySets(model, id, false);

getSpatialStructure#

IfcLoader.IfcManager.getSpatialStructure (
modelID: number
): object;

🔗🔗🔗 Gets the spatial structure of the project.

💡
空間構造は、すべてのIFCプロジェクトを編成する階層構造です（すべての物理的アイテムは、空間構造の要素に参照されます）。

関数の引数#

  • modelID IFCモデルのIDです。

例#

const model = ifcModel.mesh.modelID;
const manager = loader.ifcLoader.ifcManager;
const ifcProject = manager.getSpatialStructure(model);

🧱 Subsets#

getSubset#

IfcLoader.IfcManager.getSubset (
modelID: number,
material?: Material
): object;

🧱🔍 指定されたmaterialを持つサブセットのメッシュを取得します。素材が指定されていない場合は、元の素材を使用したサブセットが返されます。

💡
ジオメトリックサブセットとは、モデルのジオメトリを抽出したものです。例えば、特定の条件を満たすすべてのIfcDoorsとIfcWindowsを含むサブセットを抽出し、それらをハイライトしたりエクスポートしたりすることができます。

関数の引数#

  • modelID IFCモデルのIDです。

  • material (オプション) サブセットに割り当てられた素材（ある場合）。

例#

const model = ifcModel.mesh.modelID;
const manager = loader.ifcLoader.ifcManager;
const subset = manager.getSubset(model);

createSubset#

IfcLoader.IfcManager.createSubset (
config: HighlightConfigOfModel
): object;

🧱✨ 新しい幾何学的サブセットを作成します。

関数の引数#

  • config 以下のオプションを持つ設定オブジェクトです。

    • scene モデルが配置されているシーン。

    • modelID モデルのIDです。

    • ids サブセットに適合するモデルのアイテムのIDを表す。

    • removePrevious このモデルの前のサブセットをこの素材で削除するかどうか。

    • material (オプション) サブセットに適用する素材。素材が指定されていない場合、サブセットには元の素材が適用されます。

例#

const model = ifcModel.mesh.modelID;
const manager = loader.ifcLoader.ifcManager;
const config = {
modelID: model
scene: scene,
ids: [id],
removePrevious: true
}
manager.createSubset(config);
